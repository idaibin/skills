# Generate AGENTS.md

请先不要修改任何代码，先完成项目初始化理解，并输出文档草稿。

## Related Skill

- Related skill: `code-context`.
- This prompt is a standalone source asset for project-context bootstrapping.
- If its rules need to become part of the published skill, update `skills/code-context/references/` in the skill package before upgrading `code-context`.

## 语言

- 默认使用用户当前语言回复。
- 如果用户明确指定输出语言，按用户指定语言回复。
- 代码、命令、路径、API 名称、类型名和错误信息保持原文。

按以下步骤执行：

1. 识别项目基础信息
- 技术栈（前端 / 后端 / 框架 / 构建工具）
- 包管理器（pnpm / npm / yarn / bun / cargo 等）
- 运行时版本要求（node / rust 等）
- 启动方式（dev / start / serve 实际命令）

要求：
- 不要猜
- 找不到就写“未发现”

2. 梳理目录结构
请说明以下内容分别在哪，给出真实路径：
- 页面入口
- 路由层
- 组件层
- 接口 / service 层
- 状态管理
- 样式 / 主题
- 测试目录

要求：
- 用真实路径
- 没有就写“未发现”
- 不要编造结构

3. 读取并总结关键文件
读取并总结：
- README.md
- package.json
- AGENTS.md（如果存在）
- lint / typecheck / test / build 相关配置

重点提取：
- 可执行命令
- 工具链
- 开发约束
- 隐含代码习惯

4. 总结当前项目的真实开发约定
请基于仓库实际情况总结：
- 组件复用方式
- 命名风格
- 样式方案
- 接口封装方式
- 状态管理方式
- 测试习惯

要求：
- 基于代码和配置判断
- 不要写空泛的“最佳实践”

5. 尝试执行基线检查
尝试执行：
- lint
- typecheck
- test
- build

要求：
- 优先使用项目已有命令
- 不要自行编造命令

对每个命令输出：
- 使用的命令
- 是否成功
- 如果失败，说明是：
  - 环境问题
  - 配置问题
  - 项目本身已有问题

6. 生成 AGENTS.md 草稿（不要写入文件）
必须包含：
1. 仓库用途说明
2. 目录结构说明
3. 项目工作基本规则
4. 修改代码约束，重点强调：
   - 优先复用现有组件
   - 禁止无关改动
   - 保持现有风格一致
5. 改动后必须执行的检查命令
6. 完成任务后的汇报格式
7. 明确列出不允许做的事情

要求：
- 内容贴合当前仓库
- 不要写空泛规则

7. 生成 docs/project-map.md 草稿（不要写入文件）
必须包含：
1. 技术栈
2. 安装 / 启动 / 测试 / 构建命令
3. 目录结构说明
4. 页面开发通常涉及的文件链路
5. 组件、接口、状态管理、样式、测试分别在哪里
6. 高频改动区域
7. 高风险区域
8. 新任务进入时推荐阅读顺序

要求：
- 结构清晰
- 内容基于真实仓库

8. 最后输出简洁总结
包括：
- 项目结构一句话总结
- 关键命令
- 高频改动路径
- 高风险区域
- 以后我应该如何给你下任务最有效

全局要求：
- 不允许修改任何文件
- 不允许假设项目结构
- 找不到就写“未发现”
- 优先复用现有实现，不引入新模式
- 不做无关优化或重构
- 输出尽量简洁、明确、可执行
