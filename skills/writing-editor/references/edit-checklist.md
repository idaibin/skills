# Edit Checklist

Before final output, check:

- [ ] Remove low-information openings.
- [ ] Remove template summaries.
- [ ] Compress abstract phrasing.
- [ ] Preserve the original core point.
- [ ] Preserve or clarify real tradeoffs.
- [ ] Keep the author's judgment no stronger than the supplied context supports.
- [ ] Avoid mechanical parallel structure.
- [ ] Avoid adding unsupported facts.
- [ ] Preserve technical accuracy.
- [ ] Keep the tone suitable for a personal technical blog.
- [ ] For Reddit posts, use a concrete title, short opening, transparent tradeoffs, and no link-first launch voice.
- [ ] For Reddit posts, avoid claiming subreddit fit or rule compliance unless verified.
- [ ] Keep the draft publishable in the target context: the author's own blog or the specified developer community.
- [ ] Output only the edited article unless the user asked for diagnosis or notes.

If any item fails, revise before answering.
